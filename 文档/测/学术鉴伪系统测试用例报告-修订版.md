## 一、测试目的与范围

### 1.1 测试目的

对学术鉴伪系统**用户端**与**管理端**的核心业务流程进行功能性验证，覆盖用户认证、学术检测、检测历史、综合鉴伪报告、人工审核闭环及管理端运维功能；并通过后端 HTTP 接口测试验证权限边界与参数校验。

### 1.2 测试范围

**纳入范围**

- 用户端（`http://127.0.0.1:3000`）：编辑（发布者）、专家（审稿人）角色
- 管理端（`http://127.0.0.1:3001`）：组织管理员、软件管理员
- 后端 API（`http://127.0.0.1:8000/api/`）
- 图像 / 论文 PDF / Review 检测、检测历史、综合鉴伪报告、人工审核全流程
- 管理端：检测与资源、用户与组织、人工审核审批、系统日志等

**不纳入本轮结论（需在答辩材料中单独说明时引用）**

- 性能压测（3 秒上传、30 秒单任务、100 并发等指标）
- WebSocket 实时通知（未安装 `daphne` 时系统使用 HTTP 轮询，主流程仍可测）
- 深度跨模态算法融合（当前为规则层 `batch-fusion`）
- 精准模式是否调用 LLM（依赖 AI 服务配置，界面已提供标准/精准两种模式）
