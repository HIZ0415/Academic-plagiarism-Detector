## 2. 后端建议修改清单（前端不改后端，请后端实现）

以下均为 **建议**；在未实现前，前端已做 **缺省兼容**（例如任务列表缺少 `task_type` 时显示「—」）。

### 2.1 `GET /api/get_all_user_tasks/`（`views_admin.get_all_user_tasks`）

**现状**：每条任务含 `task_id`、`task_name`、`status`、`upload_time`、`completion_time`、`organization`。

**建议补充**（与统一任务模型 / Alpha 对齐）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_type` | string | 与 `DetectionTask.task_type` 一致：`image_detection`、`paper_aigc`、`resource_check`、`review_detection` |
| `user_id` | int | 任务发起人 ID |
| `username` | string | 发起人用户名（展示） |
| `error_message` | string | 失败摘要；成功可为空串；列表场景建议截断（如 500 字） |

**其它建议**：

- 未完成时 **`completion_time` 返回 `null`**，避免对空值 `localtime` 抛错。
- 查询使用 **`select_related('user', 'organization')`**，减少 N+1。
- 全局管理员按 **`organization`**（组织 ID）查询参数筛选列表的行为与概要设计一致即可。

### 2.2 `POST /api/admin-login/`（`AdminLoginView` / `AdminLoginSerializer`）

**现状**：仅当 **`user.role === 'admin'`** 时允许登录；校验失败多为 **HTTP 400** + `non_field_errors`（非 401）。

**可选改进**（体验向，非必须）：

- 密码错误与「非管理员角色」区分 **HTTP 状态码或结构化错误码**，便于前端与监控区分。
- 文档化：`createsuperuser` 用户若未改 **`role`**，默认为 `publisher`，将无法登录管理端。

### 2.3 `POST /api/register/` 与管理端「注册」

**现状**：邀请码仅绑定 **`publisher` / `reviewer`**，**不能**通过该接口得到 **`role=admin`**。

**产品语义**：管理端上的注册 = **持邀请码开通平台账号**，不是 **自助成为后台管理员**。若需要管理员自助注册，需 **新接口或流程**（例如已登录超级管理员调用 **`POST /api/create-admin/`** 等）。

详细约定见下文 **§4**。

### 2.4 PDF 报告中文字体（可选）

启动若出现 **`SimSun.ttf` 无法打开**：不影响接口与管理端登录；仅影响 **PDF 中文渲染**。可选在后端代码目录放置 `SimSun.ttf` / `SimSun-Bold.ttf`（见 `core/utils/report_generator.py` 中路径逻辑）。

## 4. 管理端登录、注册与管理员账号（交接约定）

1. **`AdminLoginView`（`POST /api/admin-login/`）**  
   - 要求 **`user.role === 'admin'`**；否则拒绝登录管理端。

2. **`InvitationCode` 与 `POST /api/register/`**  
   - 邀请码角色仅有 **`publisher` / `reviewer`**，**不能**仅靠注册接口获得 **`admin`**。

3. **管理端登录页的「注册」**  
   - 语义：**持邀请码开通平台账号**（与用户端同源 `POST /register/`），**不是**自助后台管理员注册。

4. **若需「管理员自助注册」**  
   - 需另行设计后端能力（如 `create-admin`、审批流等）。

5. **`admin@mail.com` 等账号「密码对却登不上」**  
   - 常见原因：`User.role` 仍为 **`publisher`**（如仅用 `createsuperuser` 创建）。请在库内将 **`role` 设为 `'admin'`** 并使用 **`localadmin@example.com` / 脚本生成密码** 等可预期测试账号联调。  
   - 管理端前端会解析 **`non_field_errors`**，区分密码错误与「非管理员」提示。
