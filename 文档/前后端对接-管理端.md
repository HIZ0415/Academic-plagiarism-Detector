# 前后端对接 · 管理端

本文面向 **后端**：说明管理端前端已接入的接口、**人工审核审批与发布者/专家联动**、任务监控与统计；约定 **管理端前端不在仓库内改** `views_admin.py` 等，下列契约由后端在服务端落地。

**必读总览**：[人工审核全链路、状态机、API 表](./前后端对接-总览.md) **§3**。  
**用户端分工**：[编辑模式](./前后端对接-用户端编辑模式.md)、[专家模式](./前后端对接-用户端专家模式.md)。

## 1. API 前缀

管理端 axios **`baseURL`**：`{VITE_API_URL}/api` 或本地 **`/api`**（见 [`前后端对接-总览.md`](./前后端对接-总览.md) §1）。下文路径均指 **`/api` 之后的相对段**。

## 2. 登录与注册（交接）

| 接口 | 行为 |
|------|------|
| **`POST /admin-login/`** | 仅当 **`user.role === 'admin'`** 成功；失败多为 **400** + `non_field_errors`（非 401） |
| **`POST /register/`** | 邀请码仅 **`publisher` / `reviewer`**，**不能**仅靠注册得到 **`admin`** |
| 管理端登录页「注册」 | 语义：**持邀请码开通平台账号**（与用户端同源），**不是**自助成为后台管理员 |

`createsuperuser` 若未改 **`role`**，默认可能是 **`publisher`**，将无法登录管理端；联调请使用 **`role=admin`** 的测试账号。

## 3. 人工审核申请审批（三端联动核心）

| 方法 | 路径 | 前端页面 | 说明 |
|------|------|----------|------|
| GET | `/get_reviewRequest/all/` | **`/reviews`** | 分页列表；建议每条含 **`detection_task_id`、`task_type`、`reason`**（全文或摘要） |
| GET | `/get_reviewRequest/<id>/` | 详情抽屉/页 | 同上字段 + 可选 **`imgs` / 分配信息** |
| POST | `/handle_reviewRequest/<id>/` | 通过/拒绝 | Body：`{ "choice": 1 \| 0, "reason": "拒绝必填" }`；**通过** 后必须生成专家侧可见的 **`manual_review_id`**（与 [`总览 §3`](./前后端对接-总览.md) 一致） |

前端：`代码/前端/前端管理端/src/api/review.ts`、`src/pages/reviews.vue`。

**Mock**：`VITE_USE_MOCK_MANUAL_REVIEW_WORKFLOW=true`，与用户端共用 **`代码/前端/shared/manualReviewWorkflowMock.ts`**。**注意**：用户端与管理端 dev **端口不同则 localStorage 不互通**；无后端全链路时也可仅在用户端 `/annual` 使用「模拟管理端审批」。

## 4. 工作台与导航（当前前端结构）

| 路由 | 说明 |
|------|------|
| **`/`** | 工作台：**快捷入口 + 数据看板**（原独立统计页已并入首页；访问 `/analytics` 会重定向 `/`） |
| **`/tasks`** | 检测任务监控；与 **`/files`** 顶部有 **「检测任务 ⟷ 资源与文件」** 切换条 |

| **`/members`** | **用户与组织**（Tab：用户账号 / 平台组织或本组织档案） |
| **`/reviews`** | 人工审核申请（组织管理员） |
| **`/logs`** | 操作与审计日志 |

WebSocket（可选刷新任务列表）：`ws://<host>/ws/notifications/?token=...`，见 §6。

## 5. 任务列表与可观测性（Alpha 035）

| 能力 | 接口 / 说明 |
|------|-------------|
| 列表 | **`GET /get_all_user_tasks/`**（页面 **`/tasks`**） |
| 详情 | **`GET /get_detection_task_status/<task_id>/`**（图像子结果等） |
| 轮询 | `tasks.vue`：开启「自动刷新」且存在 `pending`/`in_progress` 时定时拉列表 |
| WS | `src/composables/useAdminNotifySocket.ts` 连通知频道，防抖刷新任务列表 |

### 5.1 建议后端扩展：`GET /get_all_user_tasks/`

现有字段基础上，建议补充（与统一任务模型对齐）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_type` | string | `image_detection` / `paper_aigc` / `resource_check` / `review_detection` |
| `user_id` | int | 发起人 |
| `username` | string | 展示 |
| `error_message` | string | 失败摘要；成功可为空串 |

其它建议：未完成时 **`completion_time` 为 `null`**；查询 **`select_related('user','organization')`**；组织管理员仅本组织数据；全局管理员可按 `organization` 筛选。

## 8. 联调验收建议

1. `role=admin` 账号：`GET /get_all_user_tasks/` 字段齐全；未完成 `completion_time` 可空。
2. 组织管理员仅本组织；软件管理员按组织筛选（若已实现）。
3. **审批流**：管理端拒绝 → 发布者侧 `admin_state=refused`；通过 → 审稿人 **`GET /get_reviewer_tasks/`** 可见 **`manual_review_id`**。
4. Mock 模式下与用户端同端口验证 **`handle_reviewRequest`** 与专家池联动。
