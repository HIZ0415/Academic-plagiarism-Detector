# 前后端对接 · 用户端「编辑」（发布者 publisher）

> **文档依据**：需求规格说明书 `文档/requirements` 中 **用户展示模块**（`03-2-user-display.tex`）——检测记录、个人信息等；**用户上传模块**（`03-3-user-upload.tex`）；人工审核申请与进度见 **用户展示** 中审核任务相关条目。概要设计 **用户端路由与界面**（`overview-design/sections/06-interface-design.tex` 等）对用户端能力分层描述一致。  
> **「编辑」在用户端的含义**：登录身份为 **发布者（publisher）**，负责发起学术检测、查看历史、申请人工审核并跟踪进度；**不是**检测策略里的第三种模式（检测策略仍为 **快速 / 精准**，见 FR-YHZS-0007）。

与 **专家（审稿人）** 分工对照见 **[`前后端对接-用户端专家模式.md`](./前后端对接-用户端专家模式.md)**；全局改造索引见 **[`前后端对接-前端改造总览.md`](./前后端对接-前端改造总览.md)**。

### 协作约定（重要）

- **用户端前端迭代默认不修改后端仓库代码。**  
- 本文 **§4** 所列接口与字段由 **后端** 在服务端落地；前端仅按契约联调。  
- **界面预览（无登录盲改）**：前端可在未调用登录接口的情况下，通过本地 **界面预览** 切换「编辑 / 专家」导航（`localStorage` 键 `2-ui-preview`）；**不替代真实 JWT**，所有业务接口仍以登录后后端行为为准。

---

## 1. 概念对照

| 术语 | 含义 |
|------|------|
| **编辑（publisher）** | 用户端登录角色之一：上传与任务发起方，使用学术检测、检测历史、人工审核 **申请/跟踪**（发布者侧流程）。 |
| **专家（reviewer）** | 审稿人角色：处理人工审核 **任务池**，见专家模式文档。 |
| **快速 / 精准** | 学术检测的两种 AI 策略（上传页），与角色无关。 |

---

## 2. 前端信息架构（编辑）

| 能力 | 典型路由 | 说明 |
|------|-----------|------|
| 统一学术检测 | `/upload` | 快速/精准、多类型任务入口（论文 PDF、Review 文本等约束见总览 §8） |
| 检测记录 | `/history`、详情参数 | FR-YHZS-0002：列表、筛选、进详情、下载报告 |
| 人工审核（发布者侧） | `/annual`、结果与进度页 | 申请复核、查看进度（与 `history`、`manual-review-result` 等联动） |
| 个人中心 | `/profile` | FR-YHZS-0001：资料展示与维护（以前端已实现表单为准） |

路由守卫：**编辑** 身份不应进入审稿人专用路径（如 `/review`、`/task/...`），前端将重定向至 `/upload`（与专家互斥规则对称）。

---

## 3. 与专家模式的边界

- **编辑** 不进入「审稿任务池」列表（`/review`）；该列表仅 **reviewer** 使用。  
- **编辑** 在「人工审核」语义上指 **发起与跟踪**，专家文档中指 **处理任务**。  
- 管理端对 `ReviewRequest` 的审批（`status2` 等）见 **[`前后端对接-管理端.md`](./前后端对接-管理端.md)**。

---

## 4. 接口契约（后端实现 · 摘要）

以下路径均相对于 **`/api/`**。完整列表与优先级见 **[`前后端对接-前端改造总览.md`](./前后端对接-前端改造总览.md)** §3.1。

| 优先级 | 接口 | 说明（编辑侧） |
|--------|------|----------------|
| P0 | `POST /login/` | `role` 须为 `publisher` 且与账号一致；公开路径与 JWT 行为见专家文档 §6 |
| P0 | `GET /user-tasks/` | 检测历史列表；建议含 `task_type`、状态与时间字段 |
| P0 | `POST /detection/submit/`、`GET /tasks/...` | 图像类检测链路 |
| P0 | `POST /paper/upload/`、`POST /paper/aigc/submit/` 等 | 论文与资源检测链路 |
| P0 | 人工审核发布者侧（发起闭环） | **`POST /manual-review-requests/`**（发起）、**`GET /manual-review-requests/by-detection-task/`**（详情页进度）、**`GET /manual-review-requests/<id>/publisher-summary/`**（结果汇总）；列表沿用 **`GET /get_publisher_review_tasks/`**。完整状态机与字段见 **[`前后端对接-人工审核全流程.md`](./前后端对接-人工审核全流程.md)**。旧接口名 `create_review_task_with_admin_check`、`get_request_detail` 等若仍存在，请后端映射至上述语义。 |
| P1 | `GET /task-summary/` 等 | 个人主页统计卡片（`Profile` 页） |

## 5. 相关前端文件（索引）

| 文件 | 说明 |
|------|------|
| `src/App.vue` | 侧栏/底栏：按 **effectiveRole === publisher** 展示「学术检测」「检测历史」「人工审核申请」 |

| `src/pages/login.vue` | 编辑/专家切换、界面预览入口 |
| `src/stores/uiPreview.ts` | 无后端时的界面预览开关 |
| `src/router/index.ts` | 角色与路由互斥 |
