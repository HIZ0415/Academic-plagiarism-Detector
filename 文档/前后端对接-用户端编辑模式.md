## 2. 概念

| 术语 | 含义 |
|------|------|
| **编辑** | 用户端登录身份 **`publisher`**：发起检测、看历史、**发起人工审核申请**、看汇总结果 |
| **专家** | 指审稿人 **`reviewer`**，使用 `/review`，见专家模式文档 |
| **快速 / 精准** | 仅学术检测 AI 策略两种，与角色无关 |

**界面预览**：`stores/uiPreview` / `localStorage`，用于无登录改 UI，**不替代 JWT**。

---

## 3. 路由与守卫（与代码一致）

发布者常用路由：

| 路由 | 说明 |
|------|------|
| `/upload` | 统一学术检测入口（快速/精准）；提交后常跳转 `/history?detail_id=` |
| `/history` | 检测记录列表 + 统一详情（含人工审核进度、`manual-review-result` 入口） |
| `/detect/paper`、`/detect/review` | 论文 PDF / Review 专项；无 `task_id` 深链时守卫可重定向 `/upload`（见 `router/index.ts`） |
| `/annual` | 人工审核：**发起申请**、申请列表；Mock 下 **「模拟管理端审批」** |
| `/manual-review-result` | 汇总页，依赖 **`GET /manual-review-requests/<id>/publisher-summary/`** |
| `/profile` | 个人中心 |

**守卫**：`publisher` **不得**进入 `/review`、`/task/...`（会被重定向 **`/upload`**）。

源码索引：`src/App.vue`、`src/pages/upload.vue`、`src/pages/history.vue`、`src/pages/annual.vue`、`src/api/manualReviewWorkflow.ts`、`src/router/index.ts`。

## 4. 人工审核（发布者侧）接口 — 实现清单

**完整字段与顺序说明仅维护一份**：[`前后端对接-总览.md`](./前后端对接-总览.md) **§3.4**。此处列出 **前端实际调用封装**，便于后端对照实现。

| 前端函数 | 方法 | 路径 | 说明 |
|----------|------|------|------|
| `createManualReviewRequest` | POST | `/manual-review-requests/` | Body：`detection_task_id`、`task_type`、`reason`、`priority?`、`batch_session_id?` |
| `listPublisherManualReviewApplications` | GET | `/get_publisher_review_tasks/` | 申请列表（真实）；Mock 时走共享 Mock |
| `getManualReviewApplicationByDetectionTask` | GET | `/manual-review-requests/by-detection-task/` | Query：`detection_task_id`；供 `history` 详情刷新进度条 |
| `getPublisherManualReviewSummary` | GET | `/manual-review-requests/<id>/publisher-summary/` | 汇总页 |

文件：`代码/前端/前端用户端/src/api/manualReviewWorkflow.ts`。

### 4.1 旧接口（兼容）

`代码/前端/前端用户端/src/api/publisher.ts` 仍含：

- `POST /create_review_task_with_admin_check/`（若后端完全切换到 **`/manual-review-requests/`** 可 deprecate 并由后端返回迁移说明）
- `GET /get_request_detail/{review_request_id}/`、`GET /get_img_review_all/`、`GET /manual-review/{id}/report/` 等

新链路 **优先 §3.4 新路径**；旧路径仅迁移期映射。

## 5. 检测与任务接口（编辑侧摘要）

完整清单见 [`前后端对接-总览.md`](./前后端对接-总览.md) **§4**。
