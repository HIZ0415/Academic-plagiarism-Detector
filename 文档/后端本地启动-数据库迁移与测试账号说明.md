## 2. 每次启动后端推荐流程（给所有组员）

在后端目录执行：

```powershell
cd .\github_projects\Academic-plagiarism-Detector\代码\后端\后端代码
.\.venv\Scripts\Activate.ps1
$env:DJANGO_SETTINGS_MODULE="fake_image_detector.local_settings"
$env:AI_SERVICE_URL="http://127.0.0.1:8010"
$env:AI_SERVICE_TIMEOUT_SECONDS="120"
```

然后执行数据库相关命令（建议每次启动前都跑一遍，幂等）：

```powershell
python manage.py migrate --run-syncdb
python manage.py migrate_detection_task_v2
python manage.py backfill_detection_task_v2
```

最后启动后端：

```powershell
python manage.py runserver 127.0.0.1:8000
```

## 5. 常见坑位（请统一）

1. 不要漏设 `DJANGO_SETTINGS_MODULE`，否则可能连到远端 MySQL 而不是本地 sqlite。
2. 登录时请用“邮箱”而不是“用户名”。
3. 组员本地如果账号密码不一致，可按本文档统一重置后再测。
4. `db.sqlite3` 不要提交到 Git（已在 `.gitignore` 忽略）。
