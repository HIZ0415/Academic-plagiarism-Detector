# 云端部署前：后端数据库准备说明

本文档用于把当前项目从“本地联调数据库形态”整理为“云端可部署数据库形态”。

## 1. 本次已整理的文件

- 新增云端设置文件：`代码/后端/后端代码/fake_image_detector/cloud_settings.py`
- 新增数据库一键准备脚本：`代码/后端/后端代码/scripts/prepare_cloud_db.ps1`
- 新增云端环境变量模板：`代码/后端/后端代码/env.cloud.example`

## 2. 云端数据库建议

- 推荐数据库：MySQL 8+ / MariaDB 10+
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_unicode_ci`
- 连接账户建议最小权限（应用库读写，不给全局超级权限）

## 3. 服务器环境变量准备

以 `env.cloud.example` 为模板配置环境变量，至少包含：

- `DJANGO_SETTINGS_MODULE=fake_image_detector.cloud_settings`
- `DB_ENGINE` / `DB_NAME` / `DB_USER` / `DB_PASSWORD` / `DB_HOST` / `DB_PORT`
- `REDIS_URL`
- `AI_SERVICE_URL`

## 4. 一键初始化数据库（推荐）

在后端目录执行：

```powershell
cd 代码/后端/后端代码
.\.venv\Scripts\Activate.ps1
.\scripts\prepare_cloud_db.ps1 -DjangoSettings fake_image_detector.cloud_settings
```

该脚本会执行：

1. `manage.py check`
2. `manage.py migrate --run-syncdb`
3. `manage.py migrate_detection_task_v2`
4. `manage.py backfill_detection_task_v2`
5. `manage.py collectstatic --noinput`

## 5. 可选：初始化管理员账号（忽略）



## 6. 数据库验收检查

建议至少检查以下项：

1. `core_detectiontask` 已包含字段：`task_type`、`paper_file_id`、`error_message`
2. `core_detectiontask` 索引存在：
   - `core_detectiontask_paper_file_id_idx`
   - `dt_user_upload_idx`
   - `dt_type_status_idx`
3. 回填脚本执行后，历史数据不再依赖 task_name 推断类型
4. 后端启动无数据库连接错误，关键接口可返回 200/401（非 404）

## 7. Git 注意事项

以下仍然不应提交：

- `db.sqlite3`
- `.env`
- `local_settings.py`
- `media/`
- 运行日志与缓存文件

根目录 `.gitignore` 已覆盖上述本地产物规则，部署前执行一次 `git status` 确认无误。
