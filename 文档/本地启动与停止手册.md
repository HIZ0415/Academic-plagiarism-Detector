# 本地启动与停止手册

本文档说明本项目在 Windows 本地环境中的启动、检查、停止和重启方式。推荐优先使用脚本；如果脚本不可用，也可以按本文的手动步骤分别启动 4 个服务。

## 1. 服务端口

```text
用户端前端:   http://127.0.0.1:3000
管理端前端:   http://127.0.0.1:3001
Django 后端:  http://127.0.0.1:8000
AI HTTP 服务: http://127.0.0.1:8010
```

## 2. 推荐方式：使用脚本启动

在仓库根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-local-windows.ps1
```

脚本会自动完成：

- 检查 Python 和 Node.js 环境。
- 安装或复用本地依赖。
- 写入前端 `.env` 的 `VITE_API_URL`。
- 初始化 Django 数据库。
- 确保本地演示账号存在。
- 启动 AI、Django、用户端前端、管理端前端。

启动成功后会输出访问地址和登录账号。

## 3. 推荐方式：使用脚本停止

在仓库根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/stop-local-windows.ps1
```

如需同时清理 `.local-dev\tmp` 临时目录：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/stop-local-windows.ps1 -CleanTemp
```

## 4. 不使用脚本：首次准备

以下命令假设仓库根目录为：


```powershell
cd "$Root\代码\后端\后端代码"
```

### 4.2 前端依赖

用户端：

```powershell

& "$Root\.tools\node\node.exe" "$Root\.tools\node\node_modules\npm\bin\npm-cli.js" install
```

```powershell
& "$Root\.tools\node\node.exe" "$Root\.tools\node\node_modules\npm\bin\npm-cli.js" install
```

```powershell

Set-Content -LiteralPath "$Root\代码\前端\前端管理端\.env" -Value "VITE_API_URL=http://127.0.0.1:8000" -Encoding UTF8
```

### 4.4 AI 最小模型

如果 `detection_service\artifacts\minimal_baseline.pkl` 不存在，先训练一次：

## 6. 启动后检查

在新的 PowerShell 窗口执行：

```powershell
Invoke-RestMethod http://127.0.0.1:8010/health
(Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8000/admin/).StatusCode
(Invoke-WebRequest -UseBasicParsing http://127.0.0.1:3000/).StatusCode
(Invoke-WebRequest -UseBasicParsing http://127.0.0.1:3001/).StatusCode
```

也可以检查端口监听：

```powershell
netstat -ano | findstr ":8010 :8000 :3000 :3001"
```

## 7. 不使用脚本：手动停止

如果 4 个服务都在前台窗口运行，直接在对应窗口按 `Ctrl+C`。

如果需要按端口强制停止：

```powershell
foreach ($port in 3000,3001,8000,8010) {
    Get-NetTCPConnection -LocalPort $port -State Listen -ErrorAction SilentlyContinue |
        Select-Object -ExpandProperty OwningProcess -Unique |
        ForEach-Object { taskkill /PID $_ /T /F }
}
```

停止后确认：

```powershell
netstat -ano | findstr ":8010 :8000 :3000 :3001"
```

如果没有 `LISTENING` 输出，说明服务已经关闭。

## 8. 重启规则

修改前端代码时，Vite 通常会热更新。修改以下文件后建议重启对应前端：

```text
.env
vite.config.mts
package.json
package-lock.json
tsconfig*.json
```

修改后端普通业务代码时，Django 通常会自动重载。修改以下内容后建议重启后端：

```text
fake_image_detector/local_settings.py
fake_image_detector/settings.py
依赖安装结果
Channels / ASGI 相关配置
```

修改数据库模型后执行：

```powershell
python manage.py makemigrations
python manage.py migrate
```

修改 AI 服务代码后建议重启 AI 服务：

```text
代码/AI服务/AI服务器代码/ai_http_service.py
代码/AI服务/AI服务器代码/detection_service/*.py
代码/AI服务/AI服务器代码/detection_service/config/model_registry.json
```

修改最小基线训练逻辑后，建议重新训练并重启 AI 服务：

```powershell
python train_minimal_baseline.py
python ai_http_service.py --host 127.0.0.1 --port 8010
```

## 9. 常见问题

### 9.1 `SimSun.ttf` 字体注册失败

该提示通常不影响服务启动，只表示 PDF 报告会使用默认字体。

### 9.2 `npm.ps1` 不能运行

不要直接运行 `npm.ps1`，改用仓库内 Node 的 npm CLI：

```powershell
& "$Root\.tools\node\node.exe" "$Root\.tools\node\node_modules\npm\bin\npm-cli.js" install
```

启动 Vite 时也可以直接调用 Vite CLI：

```powershell
& "$Root\.tools\node\node.exe" .\node_modules\vite\bin\vite.js --host 127.0.0.1 --port 3000 --strictPort
```

### 9.3 后端仍尝试旧的 SSH/SCP AI 调用

确认启动 Django 前设置了：

```powershell
$env:AI_SERVICE_URL = "http://127.0.0.1:8010"
```

### 9.4 AI 超时配置不生效

本地 AI 服务读取：

```powershell
$env:AI_SERVICE_TIMEOUT_SECONDS = "120"
```

Django 调用侧同时保留：

```powershell
$env:AI_SERVICE_TIMEOUT = "1200"
```
