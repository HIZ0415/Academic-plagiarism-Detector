本文档说明当前项目在本机的启动、检查、关闭和重启规则，并同步当前 AI 侧的最小可运行方案。

## 1. 本地服务端口

```text
用户端前端：http://127.0.0.1:3000
管理端前端：http://127.0.0.1:3001
后端 Django：http://127.0.0.1:8000
AI HTTP 服务：http://127.0.0.1:8010
```

### 7.1 完整联调：异步任务、WebSocket 与轮询

**目标**：图像/论文/Review 等异步任务在「提交 → 进行中 → 完成」链路上可观测；通知至少具备 **HTTP 轮询** 或 **WebSocket** 之一与后端一致。

| 机制 | 行为说明 |
|------|----------|
| **WebSocket（用户端 / 管理端）** | 地址与 Django Channels 路由 `ws/notifications/` 对齐。配置了 `VITE_API_URL=http://127.0.0.1:8000` 时，前端直连 `ws://127.0.0.1:8000/ws/notifications/?token=…`；未配置时走 **当前页同源** `ws://127.0.0.1:3000/ws/…`，由 Vite `server.proxy['/ws']` 转发到本机 8000。可选环境变量 **`VITE_WS_URL`** 可写死完整 ws 地址。用户端在 **登录后** 自动 `Init`，登出时 `Close`。 |
| **HTTP 轮询** | 用户端 `App.vue`：已登录时定时请求 `GET /api/notification/notify/`（全栈 Mock 下间隔约 45s，后端模式下约 60s），与 WebSocket 并存作为兜底。 |
| **Mock 下异步形态** | 开启 `VITE_USE_MOCK_AIGC` 或 `VITE_USE_FULL_FRONTEND_MOCK` 时，`paper` 与 `reviewDetection` 的 **status 接口**在提交后约 2.6～2.8s 内返回 `in_progress` 与递增 `progress`，便于在「论文工作台」「Review 专项」等页验证轮询与状态机。 |

**一键脚本（可选纯前端 Mock）**：在仓库根目录执行：

```powershell
.\scripts\start-local-windows.ps1 -FullFrontendMock
```

会为**用户端** `.env` 写入 `VITE_USE_FULL_FRONTEND_MOCK=true` 且将 `VITE_API_URL` 置空（走 `/api` 相对路径；Mock 层拦截请求，可不依赖 Django）。管理端仍按脚本写入的直连后端地址联调；若仅需管理端审批 Mock，须与用户端同源或参见《前后端对接-总览》§3.5。

**后端 WebSocket 不可用时的排查**：确认使用 `manage.py runserver` 且已安装 `channels`（本仓库 `requirements.local.txt` 已含）；若仍无法建连，可改用 ASGI 服务器（如 `daphne -b 127.0.0.1 -p 8000 fake_image_detector.asgi:application`）再测 `ws://127.0.0.1:8000/ws/notifications/`。

**人工审核与角色切换的逐步操作**（含模式甲/乙、Mock 下「模拟管理端审批」）见：**[《前端联调闭环操作手册》](./前端联调闭环操作手册.md)**。

## 8. 关闭服务

### 方式一：在各自终端按 `Ctrl + C`

这是最简单的方式。

### 方式二：按端口关闭

```powershell
$ports = 3000,3001,8000,8010
foreach ($port in $ports) {
    $lines = netstat -ano | Select-String ":$port\s"
    foreach ($line in $lines) {
        $pidText = $line.ToString() -replace '.*\s+(\d+)$','$1'
        if ($pidText -and $pidText -ne "0") {
            Stop-Process -Id ([int]$pidText) -Force -ErrorAction SilentlyContinue
        }
    }
}
```

再确认：

```powershell
netstat -ano | findstr ":8010 :8000 :3000 :3001"
```

如果没有 `LISTENING` 输出，说明已关闭。

## 9. 修改代码后的重启规则

### 修改前端代码

Vite 通常会自动热更新；以下文件修改后建议重启对应前端：

```text
.env
vite.config.mts
package.json
package-lock.json
tsconfig*.json
```

### 修改后端 Python 代码

普通业务代码一般会自动重载；以下情况建议手工重启：

```text
fake_image_detector/local_settings.py
fake_image_detector/settings.py
依赖安装结果
Celery 配置
```

### 修改数据库模型

执行：

```powershell
python manage.py makemigrations
python manage.py migrate
```

然后重启后端。

### 修改 AI 服务代码

以下任意文件修改后建议重启 AI 服务：

```text
代码/AI服务/AI服务器代码/ai_http_service.py
代码/AI服务/AI服务器代码/detection_service/*.py
代码/AI服务/AI服务器代码/detection_service/config/model_registry.json
```

### 修改最小基线训练逻辑

如果修改了以下内容：

```text
代码/AI服务/AI服务器代码/train_minimal_baseline.py
代码/AI服务/AI服务器代码/detection_service/minimal_baseline.py
```

建议顺序：

1. 重新训练
2. 重启 AI 服务

对应命令：

```powershell
python train_minimal_baseline.py
python ai_http_service.py --host 127.0.0.1 --port 8010
```

## 10. 常见问题

### AI 服务能启动，但最小 profile 不工作

先确认模型文件存在：

```text
代码/AI服务/AI服务器代码/detection_service/artifacts/minimal_baseline.pkl
```

没有就重新训练：

```powershell
python train_minimal_baseline.py
```

### 后端仍尝试旧 SSH/SCP AI 调用

确保启动后端前设置：

```powershell
$env:AI_SERVICE_URL="http://127.0.0.1:8010"
```

### 超时配置不生效

当前 AI 服务读取的是：

```powershell
$env:AI_SERVICE_TIMEOUT_SECONDS="120"
```

不是旧变量 `AI_SERVICE_TIMEOUT`。

### `npm.ps1` 不能运行

改用：

```powershell
npm.cmd
```

或仓库内的：

```powershell
..\..\..\.tools\node\npm.cmd
```

## 12. 文件预处理与一键启动脚本更新

本轮后端已补充论文和专家人工 Review 的预处理流程：

- 论文检测只接收 `.pdf`。上传后后端会提取 PDF 文本，保存原始文本、清洗后文本、段落切分结果和 `paper-preprocess-v1` AI 输入。
- Review 检测只接收在线文本或 `.txt`。提交后后端会进行编码标准化、BOM 清理、控制字符清理、空白归一化，并保存 `review-preprocess-v1` AI 输入。
- `scripts/start-local-windows.ps1` 的后端依赖探针已加入 `fitz` 检查；已有虚拟环境缺少 PyMuPDF 时，可执行：

```powershell
.\scripts\start-local-windows.ps1 -ForceDependencyRefresh
```

停止脚本 `scripts/stop-local-windows.ps1` 无需为预处理流程增加额外端口，仍按本地服务端口关闭进程。
