## 启动检查

四个服务都启动后，可以在新的 PowerShell 终端中执行：

```powershell
Invoke-RestMethod http://127.0.0.1:8010/health
(Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8000/admin/).StatusCode
(Invoke-WebRequest -UseBasicParsing http://127.0.0.1:3000/).StatusCode
(Invoke-WebRequest -UseBasicParsing http://127.0.0.1:3001/).StatusCode
```

预期：

```text
AI 服务返回 status=ok
后端 admin 返回 200
用户端前端返回 200
管理端前端返回 200
```

也可以查看端口监听：

```powershell
netstat -ano | findstr ":8010 :8000 :3000 :3001"
```

## 关闭程序

### 方式一：在每个终端按 Ctrl + C

如果按上面的方式打开了 4 个 PowerShell 终端，最简单的关闭方式是在每个终端中按：

```text
Ctrl + C
```

这会停止当前终端里运行的服务。

### 方式二：按端口一次性关闭

如果不知道服务在哪个终端里，或者服务在后台运行，可以按端口关闭：

```powershell
$ports = 3000,3001,8000,8010
foreach ($port in $ports) {
    $lines = netstat -ano | Select-String ":$port\s"
    foreach ($line in $lines) {
        $pidText = $line.ToString() -replace '.*\s+(\d+)$','$1'
        if ($pidText -and $pidText -ne "0") {
            Stop-Process -Id ([int]$pidText) -Force -ErrorAction SilentlyContinue
        }
    }
}
```

关闭后可再次确认端口：

```powershell
netstat -ano | findstr ":8010 :8000 :3000 :3001"
```

如果没有 `LISTENING` 输出，说明相关服务已经关闭。

### 方式三：按 PID 关闭

如果已经知道服务 PID，可以直接关闭：

```powershell
Stop-Process -Id <PID1>,<PID2>,<PID3>,<PID4> -Force
```

PID 每次启动都会变化，因此一般不推荐把固定 PID 写死。

## 修改代码后的重启规则

### 修改前端代码

修改 `src` 下的 Vue、TypeScript、样式文件时，一般不需要重启：

```text
Vite 会自动热更新。
```

如果修改了以下文件，需要重启对应前端服务：

```text
.env
vite.config.mts
package.json
package-lock.json
tsconfig*.json
```

重启方式：

```text
在对应前端终端按 Ctrl + C，然后重新执行启动命令。
```

如果修改了 `package.json`，需要重新安装依赖：

```powershell
$env:Path="E:\grade_3_2nd\软件工程\大作业\Academic-plagiarism-Detector\.tools\node;$env:Path"
npm.cmd install
```

### 修改后端 Python 代码

修改普通 Python 代码时，Django 开发服务器通常会自动重载：

```text
一般不需要手动重启。
```

如果修改了以下内容，建议重启后端：

```text
fake_image_detector/local_settings.py
fake_image_detector/settings.py
依赖库
Celery 相关配置
```

重启方式：

```text
在后端终端按 Ctrl + C，然后重新执行后端启动命令。
```

### 修改数据库模型

如果修改了：

```text
代码/后端/后端代码/core/models.py
```

需要执行迁移：

```powershell
cd E:\grade_3_2nd\软件工程\大作业\Academic-plagiarism-Detector\代码\后端\后端代码

$env:DJANGO_SETTINGS_MODULE="fake_image_detector.local_settings"

.\.venv\Scripts\python.exe manage.py makemigrations
.\.venv\Scripts\python.exe manage.py migrate
```

然后重新启动后端服务。

### 修改 AI 桩服务代码

如果修改了：

```text
代码/AI服务/AI服务器代码/ai_http_service.py
```

需要重启 AI 服务：

```text
在 AI 服务终端按 Ctrl + C，然后重新执行 AI 服务启动命令。
```

## 常见问题

### PowerShell 提示不能运行 npm.ps1

使用 `npm.cmd`，不要直接使用 `npm`：

```powershell
..\..\..\.tools\node\npm.cmd run dev -- --host 127.0.0.1 --port 3000 --strictPort
```

### 端口被占用

查看占用端口的进程：

```powershell
netstat -ano | findstr ":3000"
```

关闭对应 PID：

```powershell
Stop-Process -Id <PID> -Force
```

### 后端没有使用本地 SQLite

启动后端前必须设置：

```powershell
$env:DJANGO_SETTINGS_MODULE="fake_image_detector.local_settings"
```

如果没有设置，Django 会使用默认 `settings.py`，默认配置会尝试连接远程 MySQL。

### 后端仍尝试旧版 SSH/SCP AI 调用

启动后端前必须设置：

```powershell
$env:AI_SERVICE_URL="http://127.0.0.1:8010"
```

否则后端可能回退到旧版 SSH/SCP AI 调用链路。

### 前端仍请求远程服务器

检查两个前端目录下的 `.env`：

```text
VITE_API_URL=http://127.0.0.1:8000
```

修改 `.env` 后需要重启对应前端服务。

## Git 注意事项

以下内容是本地运行环境或生成物，不应提交：

```text
.tools/
node_modules/
dist/
.env
db.sqlite3
local_settings.py
__pycache__/
*.pyc
*.log
```

提交前检查：

```powershell
git status --short
```

如果看到这些路径前面是 `!!`，表示已被 `.gitignore` 忽略，不会提交。

如果看到这些路径前面是 `A` 或 `M`，表示它们仍被 Git 跟踪，需要先从索引移除：

```powershell
git rm --cached -- <文件路径>
```
