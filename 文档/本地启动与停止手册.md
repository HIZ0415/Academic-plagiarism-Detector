# 本地启动与停止手册

本文档说明当前项目在本机的启动、检查、关闭和重启规则，并同步当前 AI 侧的最小可运行方案。

## 1. 本地服务端口

```text
用户端前端：http://127.0.0.1:3000
管理端前端：http://127.0.0.1:3001
后端 Django：http://127.0.0.1:8000
AI HTTP 服务：http://127.0.0.1:8010
```

## 2. 启动顺序

建议打开 4 个 PowerShell 终端：

1. AI 服务
2. 后端 Django
3. 用户端前端
4. 管理端前端

如果要做课堂内最小演示，AI 服务启动前先执行一次模型训练。

## 8. 关闭服务

### 方式一：在各自终端按 `Ctrl + C`

这是最简单的方式。

### 方式二：按端口关闭

```powershell
$ports = 3000,3001,8000,8010
foreach ($port in $ports) {
    $lines = netstat -ano | Select-String ":$port\s"
    foreach ($line in $lines) {
        $pidText = $line.ToString() -replace '.*\s+(\d+)$','$1'
        if ($pidText -and $pidText -ne "0") {
            Stop-Process -Id ([int]$pidText) -Force -ErrorAction SilentlyContinue
        }
    }
}
```

再确认：

```powershell
netstat -ano | findstr ":8010 :8000 :3000 :3001"
```

如果没有 `LISTENING` 输出，说明已关闭。

## 9. 修改代码后的重启规则

### 修改前端代码

Vite 通常会自动热更新；以下文件修改后建议重启对应前端：

```text
.env
vite.config.mts
package.json
package-lock.json
tsconfig*.json
```

### 修改后端 Python 代码

普通业务代码一般会自动重载；以下情况建议手工重启：

```text
fake_image_detector/local_settings.py
fake_image_detector/settings.py
依赖安装结果
Celery 配置
```

### 修改数据库模型

执行：

```powershell
python manage.py makemigrations
python manage.py migrate
```

然后重启后端。

### 修改 AI 服务代码

以下任意文件修改后建议重启 AI 服务：

```text
代码/AI服务/AI服务器代码/ai_http_service.py
代码/AI服务/AI服务器代码/detection_service/*.py
代码/AI服务/AI服务器代码/detection_service/config/model_registry.json
```

### 修改最小基线训练逻辑

如果修改了以下内容：

```text
代码/AI服务/AI服务器代码/train_minimal_baseline.py
代码/AI服务/AI服务器代码/detection_service/minimal_baseline.py
```

建议顺序：

1. 重新训练
2. 重启 AI 服务

对应命令：

```powershell
python train_minimal_baseline.py
python ai_http_service.py --host 127.0.0.1 --port 8010
```

## 10. 常见问题

### AI 服务能启动，但最小 profile 不工作

先确认模型文件存在：

```text
代码/AI服务/AI服务器代码/detection_service/artifacts/minimal_baseline.pkl
```

没有就重新训练：

```powershell
python train_minimal_baseline.py
```

### 后端仍尝试旧 SSH/SCP AI 调用

确保启动后端前设置：

```powershell
$env:AI_SERVICE_URL="http://127.0.0.1:8010"
```

### 超时配置不生效

当前 AI 服务读取的是：

```powershell
$env:AI_SERVICE_TIMEOUT_SECONDS="120"
```

不是旧变量 `AI_SERVICE_TIMEOUT`。

### `npm.ps1` 不能运行

改用：

```powershell
npm.cmd
```

或仓库内的：

```powershell
..\..\..\.tools\node\npm.cmd
```
