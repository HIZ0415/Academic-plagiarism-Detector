\subsubsection{功能点——提取图片（FR-TPJC-0001）}

\paragraph*{简述}

从用户上传的PDF、压缩包等非图像格式的文件中抽取图像，便于后续进行图像造假检测。该功能旨在自动化地从多种文件格式中提取图像数据，确保后续检测流程的顺利进行。通过高效的图像提取技术，系统能够快速处理用户上传的文件，提取出高质量的图像数据，为后续的AI算法检测提供基础支持。

\paragraph*{前提条件}

用户上传的文件格式为PDF、ZIP、RAR等非图像格式。

文件大小不超过系统规定的上限（如500MB）。

文件内容包含至少一张图像。

\paragraph*{主要流程}

用户上传文件：用户通过系统界面选择并上传PDF、ZIP、RAR等格式的文件。

文件格式识别：系统自动识别上传文件的格式，并调用相应的解析工具（如PDF解析库、压缩文件解压工具）。

图像数据提取：对于PDF文件，系统解析每一页内容，提取嵌入的图像数据。对压缩文件（如ZIP、RAR），系统解压文件并遍历所有子文件，提取其中的图像文件。

图像预处理：对提取的图像进行初步处理，包括格式转换（统一为JPG或PNG格式）、分辨率调整（确保图像质量适合后续检测）。

临时存储：将提取的图像保存为临时文件，并生成图像文件列表，供后续检测模块使用。

\paragraph*{后继结果}

系统输出提取的图像文件列表，包含每张图像的文件名、格式、分辨率等元数据。

图像文件列表以结构化数据形式（如JSON）存储，便于后续模块调用。

\paragraph*{补充说明}

支持的文件格式包括但不限于PDF、ZIP、RAR、TAR、7Z等常见格式。

系统需具备处理大文件的能力，能够高效解析和提取图像数据。

提取的图像需保持原始质量，避免因压缩或转换导致的信息丢失。

系统应提供错误处理机制，如文件格式不支持、文件损坏等情况下的提示信息。

\subsubsection{功能点——AI算法检测（FR-TPJC-0002）}

\paragraph*{简述}

开发基于深度学习和图像处理技术的AI算法，对提取的图像进行自动化造假检测。检测内容包括但不限于美化修饰、背景掩盖、复制伪造等常见的图像造假手段。通过先进的AI模型和算法，系统能够高效、准确地识别图像中的异常区域，帮助用户发现潜在的造假行为。

\paragraph*{前提条件}

系统已从用户上传的文件中提取出图像数据。

图像数据已完成预处理，格式统一且分辨率适合检测。

\paragraph*{主要流程}

1.图像预处理：

（1）对提取的图像进行归一化处理，统一图像尺寸和颜色空间。

（2）去除图像中的噪声（如JPEG压缩伪影）以提高检测精度。

2.特征提取与分析：

（1）使用深度学习模型（如卷积神经网络CNN）对图像进行特征提取。

（2）分析图像的局部特征（如纹理、边缘、颜色分布）以识别异常区域。

3.造假检测：

（1）美化修饰检测：

①通过分析图像的局部特征，检测是否存在高斯模糊、亮度/对比度调整等操作。

②使用图像梯度分析技术识别模糊区域。

（2）背景掩盖检测：

①使用图像修复算法（如Criminisi的inpainting方法）检测是否存在背景掩盖痕迹。

②通过纹理分析和边缘检测识别覆盖区域。

（3）复制伪造检测：

①同图复制：通过图像匹配算法（如SIFT、ORB）检测是否存在重复区域。

②重叠切割：分析图像边缘过渡，检测是否存在切割痕迹。

③跨图拼接：通过颜色直方图匹配和阴影分析，检测是否存在跨图拼接痕迹。

4.生成检测报告：

（1）标注出图像中可能存在造假的区域，并生成详细的检测报告。

（2）报告内容包括造假类型、位置、置信度评分等。

\paragraph*{后继结果}

系统输出检测报告，包含以下内容：

（1）检测图像的原图及标注图（标注出造假区域）。

（2）每个造假区域的详细描述，包括造假类型、位置、可能的影响等。

（3）检测算法的置信度评分（如0-1之间的概率值）。

检测报告以结构化数据形式（如JSON）存储，并支持可视化展示。

\paragraph*{补充说明}

检测算法需具备较高的准确性和鲁棒性，能够处理不同分辨率、不同格式的图像。

系统应支持批量检测，能够同时处理多张图像并生成综合报告。

检测报告应提供详细的文字说明，帮助用户理解检测结果。

系统应具备扩展性，便于后续添加新的检测。

\subsubsection{功能点——全篇论文AIGC检测（FR-LWJC-0001）}

\paragraph*{简述}

在用户完成论文类检测任务后，前端展示“全篇层面”的 AIGC 检测结论与摘要指标（如全文风险等级、AI 贡献占比或等价字段，以后端约定为准），使用户快速了解整篇论文的自动鉴伪概况。

\paragraph*{前提条件}

用户已登录；对应论文检测任务已创建且后端已返回可供展示的全文级结果（或返回处理中/失败状态码）。

\paragraph*{主要流程}

\begin{enumerate}
    \item 用户在检测任务详情或论文检测专页进入“AIGC 全文检测”结果区。
    \item 前端根据任务 ID 请求全文级结果接口，展示摘要卡片（结论、关键数值、更新时间等）。
    \item 若任务仍在计算中，展示进度或排队状态，并在就绪后自动或手动刷新。
    \item 用户可从摘要区跳转到段落级明细（与 FR-LWJC-0002、0003 联动，若接口已提供锚点或段落 ID）。
\end{enumerate}

\paragraph*{后继结果}

用户获得可理解的全文级结论；接口异常或超时时展示错误提示与重试入口。

\subsubsection{功能点——AI生成段落识别（FR-LWJC-0002）}

\paragraph*{简述}

前端将后端标识的疑似 AI 生成段落以列表或文中高亮等形式展示，支持用户按段落查看系统标注的位置与简要标签，便于对照原文审阅。

\paragraph*{前提条件}

论文检测任务已产生段落级标注数据；用户具备该任务的查看权限。

\paragraph*{主要流程}

\begin{enumerate}
    \item 前端拉取段落级标注列表（含段落序号、起止位置或文本片段引用等）。
    \item 在正文视图或侧栏列表中高亮对应段落；用户点击某条标注时，视图滚动至该段落。
    \item 可选展示每条标注的简短类型或置信度区间（以后端字段为准）。
\end{enumerate}

\paragraph*{后继结果}

用户可定位全部被标注段落；无标注时展示说明“未检出疑似 AI 段落”或等价文案。

\subsubsection{功能点——段落级AI生成概率分析（FR-LWJC-0003）}

\paragraph*{简述}

针对被识别或全量段落，前端展示后端给出的 AI 生成概率或分档结果（如数值、等级、热力示意），帮助用户理解模型在段落粒度上的判断强度。

\paragraph*{前提条件}

后端对相应段落返回概率或分档字段；前端具备可视化组件（表格、条形、颜色映射等）展示能力。

\paragraph*{主要流程}

\begin{enumerate}
    \item 用户在段落视图或独立“概率分析”面板中查看各段概率或分档。
    \item 支持按概率排序、筛选高于某阈值的段落（若产品启用）。
    \item 与 FR-LWJC-0002 联动：点击段落时同步显示该段概率。
\end{enumerate}

\paragraph*{后继结果}

概率数据与段落一一对应展示；缺失字段时对应单元显示“—”或隐藏该项并附说明。

\subsubsection{功能点——事实性鉴伪子结论生成（FR-LWJC-0004）}

\paragraph*{简述}

前端展示后端生成的事实性鉴伪**子结论**列表或结构化条目（如“数据矛盾”“引用不实”等类别及条目化描述），使用户了解除 AIGC 以外的事实核查维度上的发现。

\paragraph*{前提条件}

该检测任务启用了事实性鉴伪能力且后端已返回子结论结构；用户有查看权限。

\paragraph*{主要流程}

\begin{enumerate}
    \item 前端请求事实性子结论接口，按类别或严重级别分组展示。
    \item 用户展开单条子结论查看摘要；若后端提供关联原文位置或引用，前端提供跳转或悬浮展示。
\end{enumerate}

\paragraph*{后继结果}

子结论完整可读；若无子结论则展示空状态或“未触发事实性规则”类说明。

\subsubsection{功能点——事实性鉴伪原因说明（FR-LWJC-0005）}

\paragraph*{简述}

对每条事实性鉴伪子结论，前端展示后端附带的**原因说明**与依据要点（如触发的规则说明、证据片段引用等），以提升可理解性；文案内容由后端生成，前端负责排版与折叠展开。

\paragraph*{前提条件}

子结论记录中包含原因说明字段或关联证据结构。

\paragraph*{主要流程}

\begin{enumerate}
    \item 用户在子结论卡片中展开“原因说明”区域。
    \item 若存在多条证据，按列表或步骤展示；支持复制文本（若产品允许）。
\end{enumerate}

\paragraph*{后继结果}

用户能理解单条结论的判定逻辑表层信息；超长文本支持折叠与“展开全文”。

\subsubsection{功能点——检测结果可视化解释（FR-NRJC-0002）}

\paragraph*{简述}

在展示自动检测结果时，前端通过图表、高亮、时间线或关系示意等**可视化**手段，解释模型给出的关键判断依据（如段落与概率的对应关系、事实性子结论与原文位置的关联等），使用户不必仅依赖纯文本报告即可把握“为何如此判定”的脉络。具体可视化形态由产品与 UI 设计确定，数据仍来源于后端。

\paragraph*{前提条件}

后端返回足以支撑可视化的结构化数据（坐标、段落 ID、统计序列、图节点边等之一或多）；任务处于可展示状态。

\paragraph*{主要流程}

\begin{enumerate}
    \item 用户在报告或详情页进入“可视化解释”区域。
    \item 前端根据数据类型渲染对应组件（如全文—段落热力、子结论—证据链接图等）。
    \item 用户通过交互（悬停、点击、图例筛选）查看局部细节；提供导出截图或导出数据（若需求启用）。
\end{enumerate}

\paragraph*{后继结果}

可视化与文字结论一致且无歧义；数据不足以绘图时降级为表格或列表展示并提示说明。

\paragraph*{补充说明}

可视化为辅助理解手段，**最终以报告中的文字结论与后端权威字段为准**；前端不对算法结果作二次推断。
