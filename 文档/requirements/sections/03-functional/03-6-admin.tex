\subsubsection{功能点——实体添加（FR-GLWH-0001）}

\paragraph*{简述}

管理员可通过管理端界面直接添加用户、图像、论文、Review等实体，新增实体可在用户端显示或使用。

\paragraph*{前提条件}

管理员已登录系统，且待添加实体的数据符合系统规范（如格式、字段完整性）。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“实体管理”页面，选择“添加实体”。
    \item 填写实体信息（如用户账号、图像元数据、论文标题、Review内容等），上传相关文件。
    \item 系统校验数据合法性后保存至数据库。
\end{enumerate}

\paragraph*{后继结果}

新增实体成功显示在用户端对应模块。若因各种原因导致失败，系统提示错误原因并要求重试或者重新输入。

\subsubsection{功能点——实体修改（FR-GLWH-0002）}

\paragraph*{简述}

管理员可修改用户、图像、论文、Review等实体的信息，修改后同步至用户端。

\paragraph*{前提条件}

目标实体已存在于数据库中，且修改后的数据符合系统规范。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“实体管理”页面，找到目标实体
    \item 选择“编辑”操作,修改相关信息(如用户权限、图像分类标签、论文标题、Review内容等)。
    \item 提交修改后，系统更新数据库并同步至用户端。
\end{enumerate}

\paragraph*{后继结果}

用户端显示修改后的实体信息。若因各种与原因导致失败，系统提示错误原因并要求重试或者重新修改。

\subsubsection{功能点——实体删除（FR-GLWH-0003）}

\paragraph*{简述}

管理员可删除指定的用户、图像、论文、Review等实体，删除后实体不可恢复且不会在用户端显示。

\paragraph*{前提条件}

目标实体存在于数据库中且删除实体不会引发错误。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“实体管理”页面，搜索目标实体。
    \item 选择“删除”操作，系统提示确认对话框。
    \item 确认后，系统将实体标记为“已删除”，并移出用户端展示列表。
\end{enumerate}

\paragraph*{后继结果}

实体在用户端不可见且无法被检索。若删除失败，则反馈失败原因并要求重试或放弃。

\subsubsection{功能点——实体检索（FR-GLWH-0004）}

\paragraph*{简述}

管理员可通过模糊检索快速定位用户、图像、论文、Review等实体。

\paragraph*{前提条件}

管理员登录系统。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“实体管理”页面，在搜索栏输入关键词（如用户昵称、图像ID、论文标题、Review部分内容）；当对图像进行检索时，还可按照图像标签分类进行检索。
    \item 系统实时匹配数据库中的实体名称，展示结果列表。
    \item 管理员可点击条目查看详情或进行进一步操作。
\end{enumerate}

\paragraph*{后继结果}

若有匹配结果，则以条目形式呈现；若无匹配结果，系统提示“未找到相关实体”。

\subsubsection{功能点——实体分类（FR-GLWH-0005）}

\paragraph*{简述}

管理员可按学科、资源类型、造假程度等标签对图像、论文、Review进行分类管理。

\paragraph*{前提条件}

管理员登陆系统。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“实体管理”页面，选择分类标签。
    \begin{enumerate}
        \item 选择学科则按学科分类展示。同时可以进一步选择子标签（如资源类型），则可以在各学科内对图像、论文、Review进行分类。
        \item 选择资源类型则按资源类型分类展示。同时可以进一步选择子标签（如学科），则可以在各资源类型内按学科进行分类。
        \item 此外在各个最细分类内还可选择按字典序、造假程度有序排列。
    \end{enumerate}
    \item 系统提供分类情况。
\end{enumerate}

\paragraph*{后继结果}

图像、论文、Review按照分类标准进行有序展示，便于后续统计和查询。

\subsubsection{功能点——数据信息统计（FR-GLWH-0006）}

\paragraph*{简述}

管理员可查看用户、图像、论文、Review、检测任务与审核任务等维度的统计报表。

\paragraph*{前提条件}

系统已积累一定量的业务数据。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“数据统计”页面，选择统计维度（如“用户活跃度”“论文字数”“检测频率”）。
    \item 系统生成可视化图表（如柱状图、饼图）及数据表格并直接显示。
    \item 管理员可导出报表为Excel或PDF格式。
\end{enumerate}

\paragraph*{后继结果}

管理员获得直观数据统计。可导出文件保存至本地，供后续分析使用。

\subsubsection{功能点——操作记录统计（FR-GLWH-0007）}

\paragraph*{简述}

管理员可查看系统日志，包括用户操作、检测任务记录与审核任务记录等。

\paragraph*{前提条件}

管理员登录系统。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“系统日志”页面，选择日志类型（如“用户登录日志”）。
    \item 系统按时间倒序展示日志条目，支持按关键词筛选。
    \item 管理员可导出日志或标记异常事件。
\end{enumerate}

\paragraph*{后继结果}

管理员根据日志记录完成系统审计或故障排查，对异常日志进行处理。

\subsubsection{功能点——超级管理员（FR-GLWH-0008）}

\paragraph*{简述}

超级管理员可统一管理普通管理员账号与权限边界，包括新增、删除、启停和授权调整。

\paragraph*{前提条件}

当前用户为超级管理员角色，系统已启用管理员账号与权限配置功能，目标账号状态允许变更。

\paragraph*{主要流程}

\begin{enumerate}
    \item 超级管理员进入“管理员管理”页面并查看账号列表。
    \item 选择新增、删除、启停或权限调整操作并提交变更。
    \item 系统校验后更新账号状态与权限范围，记录操作日志并按需发送通知。
\end{enumerate}

\paragraph*{后继结果}

新增账号可在授权范围内使用后台；被停用或删除账号不可继续访问；权限变更立即生效且全程可追溯。失败时系统应提示原因并允许重试。

\subsubsection{功能点——用户举报审核（FR-GLWH-0009）}

\paragraph*{简述}

管理员可审核用户举报并执行处置（如警告、限制、封号），保障平台秩序与审核公正。

\paragraph*{前提条件}

举报记录已提交且未处理，当前管理员具备举报处理权限。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“举报处理”页面。
    \item 系统展示举报列表，管理员查看详情与证据并进行核实。
    \item 管理员判定成立或驳回，填写处理说明并提交。
    \item 系统更新处理状态，通知相关用户并记录日志。
\end{enumerate}

\paragraph*{后继结果}

举报形成明确结论并保留处理痕迹；相关用户收到反馈；被举报方状态按处置结果同步更新。异常时系统应提示并支持重试。

\subsubsection{\iternew{功能点——学术资源管理（FR-GLWH-0010）}}

\paragraph*{简述}

\iternew{管理员可统一管理论文、图像与 Review 资源，支持检索、分类、详情查看、状态维护与删除。}

\paragraph*{前提条件}

管理员已登录且具备资源管理权限，系统中已存在可管理的学术资源记录。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“学术资源管理”页面。
    \item 系统按类型、时间或标签展示资源，支持检索与筛选。
    \item 管理员查看资源详情并执行分类调整、状态维护或删除。
    \item 系统更新资源状态与展示结果，并记录操作日志。
\end{enumerate}

\paragraph*{后继结果}

资源分类与状态可被统一维护并与前台展示保持一致；删除资源不再参与后续流程。失败时系统应提示原因并允许重试。

\subsubsection{\iternew{功能点——论文检测日志管理（FR-GLWH-0011）}}

\paragraph*{简述}

\iternew{管理端应支持论文检测日志的记录、查询与追踪，用于任务复核和异常排查。}

\paragraph*{前提条件}

系统已生成论文检测任务及其日志，当前用户具备日志查看或管理权限。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“论文检测日志”页面。
    \item 系统按时间展示日志并支持按任务、用户、时间和状态筛选。
    \item 管理员查看任务阶段与结果摘要，必要时定位异常并导出或标记日志。
\end{enumerate}

\paragraph*{后继结果}

管理员可追踪任务执行状态并定位异常；日志可用于排障与复核。无匹配日志应提示空状态，加载失败应提示并支持重试。

\subsubsection{\iternew{功能点——模型管理与配置（FR-GLWH-0013）}}

\paragraph*{简述}

\iternew{后台应支持检测模型的统一管理与配置，包括模型信息、参数、启停状态和任务调用策略。}

\paragraph*{前提条件}

管理员已登录且具备模型管理权限，系统具备模型接入与配置持久化能力。

\paragraph*{主要流程}

\begin{enumerate}
    \item 管理员进入“模型管理与配置”页面。
    \item 系统展示模型列表及版本、适用任务、状态和参数信息。
    \item 管理员新增或编辑模型配置，调整参数、启停状态和默认调用关系。
    \item 系统完成配置校验并保存结果，记录日志并用于后续任务调用。
\end{enumerate}

\paragraph*{后继结果}

管理员可完成模型新增、编辑、启停与调用配置；新配置对后续任务生效且变更可追溯。配置不合法或权限不足时系统应阻止生效并提示原因。
