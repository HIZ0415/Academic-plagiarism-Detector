\section{\unchangedtext{概述}}

\subsection{\unchangedtext{简述}}

近十年中，国内外学术造假事件频频爆出，这些行为严重破坏了科研生态，浪费社会资源，阻碍科学技术发展，并在一定程度上打击了诚信科研工作者的积极性。随着生成式人工智能技术的快速发展，学术场景中的造假风险已不再局限于传统篡改问题，还进一步扩展到 AI 生成文本、AI 合成图像以及模板化评审等新型问题。为此，亟需建设面向多类学术内容、兼具自动检测与人工审核能力的学术 AI 鉴伪系统。该系统的建设有助于维护学术诚信，提升期刊与科研机构的审核效率，降低因 AIGC 问题引发的学术争议，并为科研活动提供更可靠的技术支撑。 

\addedtext{本项目面向学术图像、全篇论文与同行评审 Review 三类对象，构建统一的学术 AI 鉴伪平台。系统支持用户上传不同类型的学术资源，并对其是否存在 AI 生成、异常生成或造假倾向进行检测；其中，全篇论文检测需支持识别疑似 AI 生成段落，给出段落级 AI 生成概率，并输出事实性鉴伪子结论及原因说明；同行评审 Review 检测需支持识别 AI 生成或模板化倾向；学术图像检测除给出整体判定外，还需对 AI 可疑区域进行细粒度标注。系统同时支持综合鉴伪结果生成与可视化解释，提高检测结论的可理解性。}

\addedtext{除自动检测能力外，平台还应支持人工审核流程，以提升检测结果的准确性与可信度。用户可对检测任务申请人工审核，也可参与他人提交的人工审核任务并发表评论，平台对审核意见进行统计后形成最终人工审查结果汇总。系统还应支持高并发批量检测，提供稳定的多线程上传接口，实现检测状态实时同步，并支持多种格式上传。在管理端，平台需支持用户管理、学术资源管理、数据统计与日志管理，以及不同检测大模型的管理与配置，从而满足系统运维、审核监管与持续优化的需要。}

\subsection{\unchangedtext{用户群体}}

\subsubsection{\unchangedtext{普通用户}}

普通用户是系统的主要使用者，主要包括科研人员、研究生、本科生、学术期刊编辑、审稿人以及科研管理人员等。普通用户围绕“上传、检测、查看、申请复核、参与审核”五类核心活动使用系统，其主要需求如下：

\begin{itemize}
    \item \addedtext{上传学术图像、全篇论文与同行评审 Review，并获取系统返回的自动检测结果；}
    \item \addedtext{查看全篇论文中疑似 AI 生成段落、段落级 AI 生成概率、事实性鉴伪子结论及原因说明；}
    \item \addedtext{查看同行评审 Review 是否存在 AI 生成或模板化倾向，并获取相应说明；}
    \item \addedtext{查看学术图像是否存在 AI 生成或异常痕迹，并查看图像 AI 可疑位置的细粒度标注结果；}
    \item 申请人工审核，以便结合人工判断进一步确认检测结论；
    \item \addedtext{参与他人提交的人工审核任务，对图像、论文或 Review 给出审查意见并进行评论互动；}
    \item 管理个人检测记录，筛选并查看历史任务结果，下载综合鉴伪报告，并查询人工审核进度与最终结果；
    \item \addedtext{在批量检测场景下提交多份资源，实时查看检测任务状态与结果返回情况。}
\end{itemize}

\subsubsection{\unchangedtext{管理员用户}}

管理员用户负责平台的日常运行、数据维护和审核监管，具有较高的管理权限。管理员用户的主要职责如下：

\begin{itemize}
    \item 管理普通用户账户及相关信息，维护平台用户体系的安全性与可管理性；
    \item \addedtext{对平台中的学术资源进行统一管理，包括论文、图像、Review 等资源的分类、存储、检索与删除；}
    \item 管理检测任务与人工审核任务，查看任务状态、处理异常情况并维护平台运行秩序；
    \item 查看多维度统计数据，分析平台运行情况，包括用户规模、资源规模、检测任务数量、审核任务数量等；
    \item \addedtext{管理系统日志与记录，包括用户操作记录、检测任务记录，以及论文检测日志和 Review 检测日志；}
    \item 处理用户举报、违规评论或异常审核行为，并根据平台规则采取相应处置措施；
    \item \addedtext{配置和管理不同检测大模型，根据不同检测任务需求调整模型调用策略和系统参数。}
\end{itemize}

\subsubsection{\unchangedtext{超级管理员用户}}

超级管理员除具备普通管理员的全部权限外，还承担平台级权限控制与后台治理职责，主要包括：

\begin{itemize}
    \item 对普通管理员账号进行添加、删除、启停与权限分配；
    \item \addedtext{统一管理后台权限边界，保障不同角色的操作范围清晰可控；}
    \item \addedtext{对关键系统配置进行更高层级的审核与调整。}
\end{itemize}

\subsection{\unchangedtext{相关标准规范}}

平台应遵循与网络服务、数据安全、知识产权保护和学术诚信相关的法律法规与规范要求，并在系统设计与实现过程中建立相应的业务规则。主要包括：

\begin{itemize}
    \item 数据安全与个人信息保护：遵循《中华人民共和国网络安全法》《中华人民共和国数据安全法》《中华人民共和国个人信息保护法》等要求，对用户身份信息、上传内容、检测结果和日志数据采取访问控制、分级授权、加密存储、备份恢复等保护措施；
    \item 知识产权与学术资源保护：遵循《中华人民共和国著作权法》《中华人民共和国专利法》等相关要求，保护论文、图像、Review 及检测报告等学术资源的合法权益，避免未授权传播和不当使用；
    \item 网络内容发布与服务合规：遵循互联网信息服务相关管理要求，确保平台发布、展示、存储与处理的内容合法合规，不传播违法违规或明显失实的信息；
    \item \addedtext{日志留存与审计追踪：对用户操作、检测任务、人工审核任务及后台管理操作进行记录，保证关键流程可追溯、可审计，便于问题排查与系统优化；}
    \item \addedtext{权限控制与系统安全：根据普通用户、管理员和超级管理员等不同角色划分权限，防止越权访问、误操作和恶意破坏，保障平台稳定运行；}
    \item \addedtext{学术诚信与结果审慎使用：平台检测结果用于辅助识别学术内容中的 AI 生成或造假风险，系统应提供必要的解释信息，并结合人工审核机制降低误判对用户造成的不利影响。}
\end{itemize}

\subsection{\unchangedtext{系统范围}}

\subsubsection{\unchangedtext{用户通用功能}}

\begin{itemize}
    \item 注册与登录：用户可以注册、登录并维护个人基本信息；
    \item \addedtext{多类型资源上传：用户可上传学术图像、全篇论文与同行评审 Review，系统支持多种格式输入；}
    \item 自动检测与结果返回：系统对上传内容进行自动检测，并及时返回检测结果；
    \item \addedtext{全篇论文 AIGC 检测：系统识别疑似 AI 生成段落，给出段落级 AI 生成概率、事实性鉴伪子结论及原因说明；}
    \item \addedtext{同行评审 Review 检测：系统判断评审意见是否存在 AI 生成或模板化倾向；}
    \item \addedtext{学术图像检测：系统判断图像是否存在 AI 生成或异常痕迹，并标注图像中的 AI 可疑位置；}
    \item \addedtext{综合鉴伪结果展示：系统整合不同维度的分析结果，生成综合鉴伪结论与可视化解释信息；}
    \item 人工审核申请与参与：用户可申请人工审核，也可参与他人提交的人工审核任务并提交评论意见；
    \item 历史记录与报告管理：用户可查看、筛选历史检测记录，下载综合鉴伪报告，并查询人工审核进度与最终结果；
    \item \addedtext{批量上传与高并发检测：系统支持批量上传、多线程并发处理以及检测状态实时同步。}
\end{itemize}

\subsubsection{\unchangedtext{管理员功能}}

\begin{itemize}
    \item 用户管理：管理员可对普通用户信息进行查询、修改、禁用、删除等管理操作；
    \item \addedtext{学术资源管理：管理员可对论文、图像、Review 等资源进行分类、检索、存储与删除；}
    \item 检测任务与审核任务管理：管理员可查看并管理各类检测任务和人工审核任务，处理异常任务；
    \item 数据统计与分析：管理员可查看平台运行统计信息，为平台优化与资源调度提供依据；
    \item \addedtext{日志管理：管理员可查看用户操作日志、检测任务日志、论文检测日志和 Review 检测日志；}
    \item 举报与违规处理：管理员可处理举报信息，并对违规用户或异常内容采取相应措施；
    \item \addedtext{模型管理与配置：管理员可配置不同检测模型及其调用参数，支持系统按任务需求进行模型切换。}
\end{itemize}

\subsubsection{\unchangedtext{超级管理员功能}}

\begin{itemize}
    \item 管理员账号管理：超级管理员可对普通管理员进行添加、删除、启停和权限分配；
    \item \addedtext{高级配置管理：超级管理员可对关键系统配置和后台权限策略进行统一管理。}
\end{itemize}
