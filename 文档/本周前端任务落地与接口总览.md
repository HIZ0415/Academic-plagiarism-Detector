## 三、前端交付边界说明

- 本周前端已完成“接口契约 + 页面流程 + 联调入口”。
- 数据库设计与建库属于后端职责，前端通过类型约束和 API 封装保证字段一致性。
- 若后端字段变更，优先同步：
  - `src/types/core.ts`
  - `src/api/*.ts`

## 四、建议验收方式

1. 用户端：上传论文（多文件） -> 提交 AIGC/资源检测 -> 轮询状态 -> 展示结果。
2. 用户端：Review 任务列表 -> 进入详情 -> 提交审核结果。
3. 管理端：用户管理、资源管理、统计图表、日志筛选与下载分别走通一次。
4. mock 与真实后端各验证一轮，确保演示与联调都可用。
