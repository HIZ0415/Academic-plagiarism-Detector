# 前端第八周（4.20-27）交付说明

## 对齐本周任务目标

前端范围聚焦于接口对接与页面流程，不直接落库。当前已落实：

1. 论文与学术资源上传入口（支持 PDF/DOCX/TXT，多文件批量提交）。
2. 任务创建与状态同步（提交后轮询任务状态，展示进度、失败信息）。
3. 结果展示闭环（AIGC 结果摘要+段落风险，资源检测问题明细）。
4. Review 业务前端页面与接口保留并可联调（历史模块）。
5. 核心数据结构类型统一，便于后端联调与后续扩展。

## 关键文件

- `代码/前端/前端用户端/src/pages/detect/paper.vue`
  - 论文 AIGC / 学术资源双标签页。
  - 统一上传 -> 提交任务 -> 状态同步 -> 结果展示。
  - 支持 mock 与真实接口两种模式。
- `代码/前端/前端用户端/src/api/paper.ts`
  - 论文上传、AIGC 提交、资源检测提交、任务状态、结果查询接口封装。
  - 支持 upload+submit 一步式调用，便于批量任务处理。
- `代码/前端/前端用户端/src/types/core.ts`
  - 核心实体类型：用户、资源文件、检测任务、AIGC 结果、资源检测结果、Review、通知、日志。

## 接口清单（前端侧）

- `POST /paper/upload/`
- `POST /paper/aigc/submit/`
- `GET /paper/tasks/{taskId}/status/`
- `GET /paper/aigc/{taskId}/result/`
- `POST /paper/resource-check/submit/`
- `GET /paper/resource-check/{taskId}/result/`

Review 相关（已有模块）：

- `GET /get_reviewer_tasks/`
- `GET /get_publisher_review_tasks/`
- `GET /get_review_request_detail/{manual_review_id}/`
- `POST /post_review/{manual_review_id}/`

## 联调建议

1. 后端优先保证 `task_id`、`status`、`error_message` 字段稳定。
2. AIGC 结果建议稳定返回 `ai_contribution_ratio`、`summary`、`paragraphs`。
3. 资源检测结果建议稳定返回 `suspected_risk_count`、`summary`、`issues`。
4. 若后端字段有变更，只需在 `paper.ts` 和 `types/core.ts` 同步即可。
