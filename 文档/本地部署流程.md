# 本地部署流程

本文档说明如何在本机部署当前项目，并重点补充当前 AI 侧的最小可运行演示链路。

## 1. 项目组成

- 后端：`代码/后端/后端代码`
- 用户端前端：`代码/前端/前端用户端`
- 管理端前端：`代码/前端/前端管理端`
- AI 服务：`代码/AI服务/AI服务器代码`

## 2. 推荐本地访问入口

```text
用户端前端：http://127.0.0.1:3000
管理端前端：http://127.0.0.1:3001
后端 API：http://127.0.0.1:8000/api/
AI 服务健康检查：http://127.0.0.1:8010/health
AI 只读管理接口：http://127.0.0.1:8010/api/v1/admin/model-registry
```

### 2.1 一键启动与完整联调

仓库根目录 `scripts/start-local-windows.ps1` 可拉起 AI、Django、双前端；可选 `-FullFrontendMock` 启用用户端全栈 Mock。异步任务、通知 **WebSocket** 与 **HTTP 轮询** 的约定与验证步骤见《[本地启动与停止手册](./本地启动与停止手册.md)》**§7.1**。

## 3. 环境要求

建议环境：

- Python 3.11 或 3.12
- Node.js 20+
- Redis 6+
- MySQL 8 或 MariaDB 10+

AI 侧最小演示额外依赖：

- `numpy`
- `pillow`
- `opencv-python`
- `scikit-learn`

注意：

- 当前仓库中**不应假定存在可直接复用的后端 `.venv`**
- 如果要跑“AI 训练 + AI 服务最小演示”，请确保 AI 运行环境包含 `scikit-learn`

## 4. 后端部署

进入后端目录：

```powershell
cd E:\grade_3_2nd\软件工程\大作业\Academic-plagiarism-Detector\代码\后端\后端代码
```

创建虚拟环境并安装依赖：

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.local.txt
```

静态检查：

```powershell
python manage.py check
```

## 5. 数据库初始化

如使用本机 MySQL，先创建数据库：

```sql
CREATE DATABASE fake_image_detector_db_beta DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后根据实际环境修改后端数据库配置，再执行：

```powershell
python manage.py makemigrations
python manage.py migrate
python manage.py createsuperuser
```

## 6. 启动 Redis

使用 Docker：

```powershell
docker run --name academic-redis -p 6379:6379 -d redis:7
```

或直接启动本机 Redis 服务，确保监听 `127.0.0.1:6379`。

## 7. AI 服务部署

进入 AI 服务目录：

```powershell
cd E:\grade_3_2nd\软件工程\大作业\Academic-plagiarism-Detector\代码\AI服务\AI服务器代码
```

### 7.1 创建 AI 虚拟环境

推荐单独创建 AI 环境：

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install numpy pillow opencv-python scikit-learn requests
```

说明：

- 默认重模型 profile 还可能依赖更重的推理环境
- 课堂最小演示只要求跑通 `minimal_trainable` profile，上述依赖足够

### 7.2 训练最小可运行模型

执行：

```powershell
python train_minimal_baseline.py
```

训练成功后会生成：

```text
代码/AI服务/AI服务器代码/detection_service/artifacts/minimal_baseline.pkl
代码/AI服务/AI服务器代码/detection_service/artifacts/minimal_baseline_eval.json
代码/AI服务/AI服务器代码/detection_service/artifacts/minimal_dataset/
```

其中：

- `minimal_baseline.pkl`：最小可运行模型
- `minimal_baseline_eval.json`：测试集输出结果
- `minimal_dataset/`：自动生成的训练/测试图片

### 7.3 启动 AI HTTP 服务

```powershell
python ai_http_service.py --host 127.0.0.1 --port 8010
```

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8010/health
```

当前典型返回会包含：

```json
{
  "status": "ok",
  "service_version": "ai-detection-service-2026-04",
  "supported_tasks": ["image"],
  "reserved_tasks": ["paper", "review"],
  "result_format": "standard-evidence-v1",
  "available_image_profiles": ["default", "fast", "minimal_trainable"]
}
```

查看只读管理接口：

```powershell
Invoke-RestMethod http://127.0.0.1:8010/api/v1/admin/model-registry?profile=minimal_trainable
```

## 11. 常见问题

### AI 服务启动了，但最小 profile 不可用

先检查是否已训练：

```powershell
python train_minimal_baseline.py
```

再检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8010/api/v1/admin/model-registry?profile=minimal_trainable
```

### 后端仍走旧 SSH/SCP 调用链路

启动后端前必须设置：

```powershell
$env:AI_SERVICE_URL="http://127.0.0.1:8010"
```

### AI 请求超时

当前 AI 服务读取的超时环境变量是：

```powershell
$env:AI_SERVICE_TIMEOUT_SECONDS="120"
```

不是旧写法 `AI_SERVICE_TIMEOUT`。

### 前端端口冲突

用户端使用 `3000`，管理端使用 `3001`，并加 `--strictPort`。

### 文件预处理依赖缺失

论文预处理依赖 `PyMuPDF`，代码中导入名为 `fitz`。如果上传 PDF 时出现 `ModuleNotFoundError: No module named 'fitz'`，请在后端虚拟环境中重新安装本地依赖：

```powershell
cd 代码/后端/后端代码
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.local.txt
```

当前文件预处理范围保持为代码已覆盖格式：

- 论文：仅 `.pdf`，流程为 PDF 文本提取、清洗、段落切分、生成 `paper-preprocess-v1` AI 输入。
- 专家人工 Review：在线文本或 `.txt`，流程为解码与编码标准化、BOM 清理、控制字符清理、空白归一化、生成 `review-preprocess-v1` AI 输入。
