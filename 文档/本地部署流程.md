# 本地部署流程

本文档用于在本机验证当前项目代码的可行性。当前项目由四部分组成：

- 后端：`代码/后端/后端代码`，Django + Django REST Framework + Channels + Celery。
- 用户端前端：`代码/前端/前端用户端`，Vue 3 + Vite。
- 管理端前端：`代码/前端/前端管理端`，Vue 3 + Vite。
- AI 推理服务：`代码/AI服务/AI服务器代码`，当前提供 HTTP 桩服务 `ai_http_service.py`，可用于本地联调。

## 推荐本地部署路径

项目根目录：

```text
E:\grade_3_2nd\软件工程\大作业\Academic-plagiarism-Detector
```

最短可行路径：

1. 启动 AI HTTP 桩服务，监听 `http://127.0.0.1:8010`。
2. 启动 Redis，用于 Django Channels 和 Celery。
3. 启动后端 Django 开发服务器，监听 `http://127.0.0.1:8000`。
4. 启动用户端前端，监听 `http://127.0.0.1:3000`。
5. 启动管理端前端，监听 `http://127.0.0.1:3001`，避免和用户端端口冲突。

本地访问入口：

```text

管理端：http://127.0.0.1:3001
后端 API：http://127.0.0.1:8000/api/
AI 服务健康检查：http://127.0.0.1:8010/health
```

## 环境要求

必需软件：

- Python 3.11 或 3.12。当前仓库内后端 `.venv` 使用的是 Python 3.12.1。
- Node.js 20 或 22，需包含 `npm`。
- Redis 6+。
- MySQL 8 或 MariaDB 10+。

后端当前代码的 `fake_image_detector/settings.py` 中数据库配置仍写死为远程 MySQL：

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'fake_image_detector_db_beta',
        'USER': 'lzy',
        'PASSWORD': 'root123',
        'HOST': '122.9.45.122',
        'PORT': '3306',
    }
}
```

如果只做快速可行性验证，可以暂时使用现有配置；如果要完全本地部署，需要把 `HOST`、`USER`、`PASSWORD`、`NAME` 改成本机数据库配置，或后续将这些配置改造成环境变量。

## 后端部署

进入后端目录：

```powershell
cd E:\grade_3_2nd\软件工程\大作业\Academic-plagiarism-Detector\代码\后端\后端代码
```

### 方式一：使用仓库内已有虚拟环境

当前仓库已经包含后端虚拟环境 `.venv`，可直接用于本机验证：

```powershell
.\.venv\Scripts\Activate.ps1
python --version
python manage.py check
```

当前已验证结果：

- `python --version` 返回 `Python 3.12.1`。
- `python manage.py check` 可以通过系统检查。
- 检查时有 1 个 Django URL 警告：`/review-requests/<int:review_request_id>/delete/` 路由开头多了 `/`，不阻塞本地启动。
- 检查时提示缺少 `SimSun.ttf`，PDF 报告会使用默认字体，不阻塞后端启动。

### 方式二：重新创建虚拟环境

如果本机不使用仓库内 `.venv`，可重新创建：


建议后续补充正式的 `requirements.txt`，避免依赖版本只存在于本机虚拟环境中。

### 数据库初始化

如果使用本机 MySQL，先创建数据库：

```sql
CREATE DATABASE fake_image_detector_db_beta DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后修改 `fake_image_detector/settings.py` 中 `DATABASES` 为本机连接信息。

执行迁移：

```powershell
python manage.py makemigrations
python manage.py migrate
python manage.py createsuperuser
```

当前 `core` 目录没有提交 `migrations` 目录，因此第一次本地部署时需要执行 `makemigrations`。

### 启动 Redis

Windows 本机可用 Docker 启动 Redis：

```powershell
docker run --name academic-redis -p 6379:6379 -d redis:7
```

如果已安装本机 Redis，也可以直接启动本机 Redis 服务，确保端口为 `6379`。

### 启动后端

推荐在本地显式指定 AI HTTP 服务地址：

```powershell
$env:AI_SERVICE_URL="http://127.0.0.1:8010"
$env:AI_SERVICE_TIMEOUT="1200"
python manage.py runserver 127.0.0.1:8000
```

如需运行 Celery：

```powershell
celery -A fake_image_detector.celery worker -Q ai --pool=solo --loglevel=info
```

另开一个终端运行 CPU 队列：

```powershell
celery -A fake_image_detector.celery worker -Q cpu --pool=solo --loglevel=info
```

## 常见问题

### 两个前端端口冲突

用户端和管理端的 `vite.config.mts` 都配置了 `server.port = 3000`。同时启动时，管理端使用：

```powershell
npm run dev -- --host 127.0.0.1 --port 3001
```

### 前端仍请求远程服务器

检查两个前端目录下的 `.env`：

```text
VITE_API_URL=http://127.0.0.1:8000
```

修改后需要重启 Vite。

### 后端仍尝试旧版 SSH/SCP AI 调用

启动后端前必须设置：

```powershell
$env:AI_SERVICE_URL="http://127.0.0.1:8010"
```

否则后端会回退到 `core/call_figure_detection.py` 中的旧版 SSH/SCP 调用链路。

### Redis 未启动

Channels 和 Celery 默认连接：

```text
127.0.0.1:6379
```

如果没有 Redis，WebSocket 通知和 Celery 任务会失败。

### PDF 报告字体提示

`python manage.py check` 中出现 `SimSun.ttf` 缺失提示时，后端会使用默认字体，通常不影响启动。若需要中文 PDF 效果稳定，可把宋体字体文件放到后端目录并保持文件名为 `SimSun.ttf`。
