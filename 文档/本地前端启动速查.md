# 本地前端启动速查（每次开机照着做）

## 1. 启动前确认

- 已安装 Node.js（LTS）
- 能在终端执行：

```powershell
node -v
npm -v
```

如果 `npm` 报“无法识别”，先安装/重装 Node.js 并勾选 `Add to PATH`，然后重开终端。

---

## 2. 启动用户端（推荐先开这个）

```powershell
cd "g:\大学\大三下\软工\Academic-plagiarism-Detector\代码\前端\前端用户端"
npm install
npm run dev
```

启动成功后看终端输出，找到：

- `Local: http://localhost:3000/` 或
- `Local: http://localhost:3001/`

登录页地址就是：

- `http://localhost:端口/login`

---

## 3. 启动管理端（可选）

```powershell
cd "g:\大学\大三下\软工\Academic-plagiarism-Detector\代码\前端\前端管理端"
npm install
npm run dev
```

如果提示 `Port 3000 is in use, trying another one...`，说明端口冲突，Vite 会自动换端口（通常 3001）。

---

## 4. 常见问题速解

## 4.1 浏览器报 `ERR_CONNECTION_REFUSED`

原因：前端服务没启动或终端被关了。  
处理：

1. 回到前端终端窗口看是否还在运行  
2. 如果终端已关闭，重新执行 `npm run dev`  
3. 按终端显示的 `Local` 地址访问

## 4.2 `npm` 命令找不到

原因：Node.js 未安装或 PATH 未生效。  
处理：安装 Node.js（LTS）并重开终端。

## 4.3 页面能开但接口失败

先检查 `.env` 是否存在且正确：

- `代码/前端/前端用户端/.env`
- `代码/前端/前端管理端/.env`

内容应为：

```env
VITE_API_URL=http://114.116.204.131
```

改完后必须重启 `npm run dev`。

## 5. 我当前项目的常用地址（本地开发）

- 用户端：`http://localhost:3001/login`（若用户端跑在 3001）
- 管理端：`http://localhost:3000/login`（若管理端跑在 3000）

注意：实际端口以终端输出为准，不要死记 3000/3001。

---

## 6. 最简流程（30秒版）

1. 打开终端进入用户端目录  
2. `npm run dev`  
3. 复制终端 `Local` 地址到浏览器  
4. 打开 `/login`  
5. 不要关终端窗口
